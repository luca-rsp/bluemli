<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cloudflare edge → visitor browser | Headers in `_headers` are applied at the edge by Workers Static Assets BEFORE the response reaches the visitor. |
| browser → Umami Cloud | The CSP `connect-src` directive gates outgoing POSTs from the page to Umami's events endpoint. |
| `public/_headers` (build input) | Static text file; not executable code; no input sanitization risk. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Tampering | Protocol downgrade (HTTPS → HTTP MITM) on subsequent visits | mitigate | `Strict-Transport-Security: max-age=63072000; includeSubDomains; preload` — 2y duration, `includeSubDomains` so future subdomains inherit, `preload` directive so the header is preload-list eligible. **Submission to hstspreload.org is deferred to v1.x** (CONTEXT.md Deferred Ideas) — the header alone is browser-cached, ad-hoc-revocable; preload-list submission is the one-way step we're not yet taking. |
| T-04-07 | Tampering / Information Disclosure | Clickjacking via iframe embedding | mitigate | CSP `frame-ancestors 'none'` (sets X-Frame-Options:DENY behavior in modern browsers). The marketing site has no legitimate embed use case. |
| T-04-08 | Spoofing | MIME-sniffing (browser interprets a `.png` as `.html`) | mitigate | `X-Content-Type-Options: nosniff`. |
| T-04-09 | Information Disclosure | Referrer leakage to third-party destinations (Instagram clicks, mailto fallback opening an external mail client) | mitigate | `Referrer-Policy: strict-origin-when-cross-origin` — origin-only on cross-origin nav, full path on same-origin. |
| T-04-10 | Tampering | Powerful-API misuse (camera, microphone, geolocation, payment, USB) | mitigate | `Permissions-Policy: accelerometer=(), camera=(), geolocation=(), gyroscope=(), magnetometer=(), microphone=(), payment=(), usb=(), interest-cohort=(), browsing-topics=()` — empty allowlists disable for all origins including self (per W3C Permissions-Policy spec). |
| T-04-11 | Tampering | XSS via inline event handlers or eval | mitigate | CSP `script-src 'self' https://cloud.umami.is` — no `'unsafe-eval'`, no `'unsafe-inline'` for scripts. Only Umami's `<script src=>` is allowlisted. |
| T-04-12 | Information Disclosure | Umami events endpoint origin drift breaking CSP `connect-src` (visible only in DevTools, looks like Pitfall #19) | mitigate | `connect-src 'self' https://cloud.umami.is https://*.umami.is https://*.umami.dev` — wildcards cover the family of origins Umami has used over the last 2 years (per RESEARCH.md Pitfall 1, sources verified to 2026). EMPIRICAL VERIFICATION at Plan 05 launch-checklist item 5: click each event, check DevTools Network for a 200/204 POST to a `*.umami.*` origin; if blocked, widen to `https:` and document tightening as v1.x. |
| T-04-13 | Tampering | `<base>` injection redirecting relative URLs to attacker origin | mitigate | CSP `base-uri 'self'`. |
| T-04-14 | Tampering | Form-submission hijacking to off-site endpoint | mitigate | CSP `form-action 'self'` (defense in depth; v1 has no forms but the directive costs nothing). |
| T-04-15 | Tampering | `_headers` file has no effect on Worker-generated responses | accept | v1 ships ZERO `/api/*` routes (per Phase 3 D-22; the `run_worker_first: ["/api/*"]` in `wrangler.jsonc` is reserved for a future contact form). So `_headers` covers 100% of v1 traffic. The `_headers` file's comment block documents this explicitly so a future engineer adding `/api/*` knows to also inject headers Worker-side. |
| T-04-16 | Information Disclosure | FLoC / Topics fingerprinting | mitigate | `Permissions-Policy: interest-cohort=(), browsing-topics=()` (defense-in-depth; the headers are honored by Chrome even though Topics is opt-in now). |
| T-04-17 | Information Disclosure | Cache-poisoning replay on fingerprinted Astro assets | mitigate | `/_astro/*` ships `Cache-Control: public, max-age=31536000, immutable` — content-hashed assets cannot collide; `immutable` tells browsers not to revalidate. |
</threat_model>

```
# Cloudflare Workers Static Assets headers file — Phase 4 LCH-04 / D-08 / D-09.
# Applies to every static-asset response. Does NOT apply to Worker code (v1 ships
# no /api/* routes; run_worker_first: ["/api/*"] in wrangler.jsonc is RESERVED but
# unrouted — if a future phase adds /api/*, that Worker MUST inject these headers
# in its own response, because _headers is NOT consulted for Worker responses).
#
# Order: most-specific path first; '/*' last. Cloudflare reads every block; more
# specific paths can override the broad block on the same header name.
# Reference: https://developers.cloudflare.com/workers/static-assets/headers/

# --- Fingerprinted Astro assets (CSS, JS, fonts emitted by Vite/Astro build) ---
# Content-hashed → safe to cache forever.
/_astro/*
  Cache-Control: public, max-age=31536000, immutable

# --- Static images with stable filenames ---
# 7 days = founder can refresh a hero on a PR-preview-then-merge loop and not wait
# too long for stale caches. (RESEARCH.md A6: CONTENT_EDITING.md convention is
# "never delete, flip availability" — pieces get a new slug, not a re-shoot at the
# same slug — so 7d is safe.)
/gallery/*
  Cache-Control: public, max-age=604800
/og-default.png
  Cache-Control: public, max-age=604800
/mark.svg
  Cache-Control: public, max-age=604800
/favicon.svg
  Cache-Control: public, max-age=604800
/favicon-16.png
  Cache-Control: public, max-age=604800
/favicon-32.png
  Cache-Control: public, max-age=604800
/favicon.ico
  Cache-Control: public, max-age=604800
/apple-touch-icon.png
  Cache-Control: public, max-age=604800

# --- Everything else (HTML, robots.txt, sitemap-*.xml) — always fresh ---
# Cloudflare's edge caches at its own discretion; we just tell browsers not to.
/*
  Cache-Control: public, max-age=0, must-revalidate
  Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
  X-Content-Type-Options: nosniff
  Referrer-Policy: strict-origin-when-cross-origin
  Permissions-Policy: accelerometer=(), camera=(), geolocation=(), gyroscope=(), magnetometer=(), microphone=(), payment=(), usb=(), interest-cohort=(), browsing-topics=()

```
