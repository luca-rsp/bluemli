<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → cloud.umami.is | Visitor's browser POSTs analytics events over HTTPS to Umami Cloud. No PII transmitted (cookieless tracker). |
| build env → static HTML | `PUBLIC_UMAMI_WEBSITE_ID` and `PUBLIC_CF_WORKERS_URL` are inlined into every page's HTML at build time. Public identifiers, not secrets. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | Umami `<script>` tag in HTML | accept | `PUBLIC_UMAMI_WEBSITE_ID` is the public identifier for the Umami site (designed to be visible). Not a secret; not subject to leak risk. Documented in `SETUP-DNS.md` (Plan 03) as "Plaintext, not Secret." |
| T-04-02 | Spoofing | Forged analytics events from non-production origins | mitigate | `data-domains` attribute makes Umami drop POSTs from any hostname not in the allowlist. Production list = apex; pre-cutover list = apex + active workers.dev hostname. No third-party can spoof events into the founder's dashboard without controlling one of those domains. |
| T-04-03 | Tampering | Umami CDN supply-chain (script.js modified by attacker) | accept | Only ONE third-party script (`cloud.umami.is/script.js`) is loaded. CSP `script-src` (Plan 02) restricts to that origin. SRI hashes are NOT pinned because the URL is versionless — pinning would break analytics on every Umami release (deferred to v1.x per CONTEXT.md). Risk is bounded because Umami is the only third-party JS the site loads. |
| T-04-04 | Information Disclosure | Per-piece slug transmitted as event metadata (`data-umami-event-piece={slug}`) | accept | Slug is non-PII (it's the public URL slug already visible in the browser address bar and sitemap-0.xml). Founder approved the per-piece breakdown via D-01. |
| T-04-05 | Repudiation | Visitor disables JS or blocks `cloud.umami.is` | accept | No mitigation needed — analytics blocking is the visitor's right. Founder accepts that some traffic will not appear in Umami. Cookieless model is the entire point of choosing Umami over Plausible (paid) or GA (consent banner). |
</threat_model>
