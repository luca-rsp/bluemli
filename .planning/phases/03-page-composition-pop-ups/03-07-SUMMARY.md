# Phase 03 Plan 07: About Page Heart Glyph Fix Summary

One-line: Added `filled={false}` to `Mark.Heart` in About.jsx signature, fixing the outline-vs-filled heart brand defect (GAP-04 / WR-06).

## What Was Done

Closed GAP-04 (WR-06 / SC3 / PAG-05): the About-page signature was rendering the FILLED heart `♥` (U+2665 BLACK HEART SUIT) instead of the OUTLINE heart `♡` (U+2661 WHITE HEART SUIT) specified in D-16 and the UI-SPEC Copywriting Contract (`"made with love from NOPA ♡"`).

**Root cause:** `Mark.Heart` in `src/components/design-skill/Mark.jsx` has a default of `filled = true`, which emits `♥`. The About.jsx invocation `<Mark.Heart color="var(--coral-500)" />` passed no `filled` prop, so the default fired.

**Fix:** Added `filled={false}` explicitly to the Mark.Heart invocation on line 50 of About.jsx.

## File Changed

`src/components/design-skill/About.jsx` — line 50

Before:
```jsx
made with love from NOPA <Mark.Heart color="var(--coral-500)" />
```

After:
```jsx
made with love from NOPA <Mark.Heart color="var(--coral-500)" filled={false} />
```

All other lines (surrounding div styles, text content, Mark import) are unchanged.

## Chosen Fix Path

Used the **`filled={false}` prop** on `Mark.Heart`, not the inline-span alternative:
- Preserves the `Mark` abstraction — all heart glyphs in the codebase flow through `Mark.Heart`
- Smaller diff (one prop added vs. removing component usage and inlining a span)
- Consistent with design skill conventions and D-16 ("existing signature treatment is `<Mark.Heart />`")
