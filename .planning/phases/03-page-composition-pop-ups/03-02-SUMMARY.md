# Phase 03 Plan 02: SEO, Sitemap, Robots Summary

Shared `<SEO />` component + env-aware `/robots.txt` + `@astrojs/sitemap` integration + 1200x630 og-default PNG, plus the REQUIREMENTS/PROJECT/ROADMAP doc edits that finalize the no-form scope cut.

## What Was Built

### `src/lib/site-url.ts` — three build-time helpers
- `resolveCanonicalBase(astroSite?: URL): string` — ALWAYS returns apex (`https://studiobluemli.com`), never consults preview env vars. Used by `<link rel="canonical">` and `<meta property="og:url">`. Concern 1 fix.
- `resolveAssetBase(astroSite?: URL): string` — preview-aware. Precedence: `CF_PAGES_URL` → `CF_WORKERS_URL` → `PUBLIC_SITE_URL` → canonical. Used by per-piece og:image overrides where preview-deploy unfurls matter.
- `isProduction(): boolean` — Concern 7 hardening: checks `process.env.WORKERS_CI_BRANCH`, `process.env.PUBLIC_DEPLOY_ENV`, `import.meta.env.WORKERS_CI_BRANCH`, `import.meta.env.PUBLIC_WORKERS_CI_BRANCH`, `import.meta.env.CF_PAGES_BRANCH`, `import.meta.env.PUBLIC_DEPLOY_ENV`. Default `false` in local dev → safer `Disallow: /` robots default.

### `src/components/SEO.astro` — per-page meta emitter (9 head elements)
Emits: `<meta name="description">`, `<link rel="canonical">`, `<meta property="og:title">`, `<meta property="og:description">`, `<meta property="og:image">`, `<meta property="og:url">`, `<meta property="og:type">`, `<meta name="twitter:card">`, `<meta name="twitter:image">`.

Does NOT emit the page title tag (Concern 4 reconciliation): `BaseLayout.astro` still owns `<title>{title}</title>` via its existing `title` prop. This is the least-blast-radius mid-phase change.

Props: `title?` (optional override for og:title), `description?` (falls back to `site.og_description`), `ogImage?` (falls back to `${canonicalBase}/og-default.png`), `pathname?` (defaults to `/`).

### `src/pages/robots.txt.ts` — env-aware endpoint
`export const prerender = true` so Astro emits to `dist/client/robots.txt` at build (Pitfall 4 avoidance). Branches body on `isProduction()`:
- Production main: `User-agent: *\nAllow: /\nSitemap: https://studiobluemli.com/sitemap-index.xml\n`
- Preview / local dev: `User-agent: *\nDisallow: /\n`

### `scripts/generate-og-default.mjs` + `public/og-default.png`
One-shot generator that rasterizes `assets/logo/mark-coral.svg` onto a 1200x630 cream-100 (`#F5DCC7`) canvas via sharp; output committed at `public/og-default.png`. Idempotent (verified — re-run produced byte-identical MD5). Wired to `npm run og:default`.

### `astro.config.mjs` — sitemap integration
`integrations: [react(), sitemap()]` — auto-discovers all prerendered routes. Resulting sitemap includes 11 entries (5 top-level routes: `/`, `/about/`, `/gallery/`, `/popups/`, `/say-hi/` + 6 gallery slugs), all canonical to apex.

### Gallery page refactor
- `src/pages/gallery.astro`: adds `<SEO slot="head" title="Gallery — Studio Bluemli" pathname="/gallery" />`.
- `src/pages/gallery/[slug].astro`: deletes the inline 4-source `ogBase` block, swaps in `resolveAssetBase(Astro.site)`, replaces inline `<meta slot="head" property="og:image">` with `<SEO slot="head" title={...} description={description} ogImage={ogImageUrl} pathname={`/gallery/${slug}`} />`.

### Doc edits (Concern 6 — bundled here, not split)
