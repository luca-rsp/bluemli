## Pattern Assignments

### `astro.config.mjs` (config, build-time)

**Analog:** RESEARCH.md lines 886–942 (verified against Astro 6 + Cloudflare adapter 13.5 docs, May 2026).

**Imports pattern (lines 887–889):**
```js
import { defineConfig, passthroughImageService, fontProviders } from 'astro/config';
import cloudflare from '@astrojs/cloudflare';
import react from '@astrojs/react';
```

Note: `fontProviders` from `astro/config` (NOT `astro:fonts`). `<Font />` component (used in BaseLayout) imports from `astro:assets` — different module. This split is a known Pitfall #8 footgun.

**Core config (lines 891–941):**
```js
export default defineConfig({
  site: 'https://studiobluemli.com',
  output: 'static',
  adapter: cloudflare({}),
  integrations: [react()],
  image: { service: passthroughImageService() },   // Sharp doesn't run in workerd — Pitfall #9
  fonts: [
    { provider: fontProviders.google(), name: 'Bagel Fat One', cssVariable: '--font-wordmark-loaded', weights: [400], display: 'swap' },
    { provider: fontProviders.google(), name: 'Caveat Brush',   cssVariable: '--font-display-loaded',  weights: [400], display: 'swap' },
    // SWAP PATH (D-17): replace Caveat Brush entry with `provider: fontProviders.local()` pointing at public/fonts/<file>.woff2 when founder provides one
    { provider: fontProviders.google(), name: 'Nunito',         cssVariable: '--font-body-loaded',     weights: [400, 700], display: 'swap' },
    // --font-hand (Caveat) is OPTIONAL per D-16 — uncomment only if a component references it
  ],
});
```

**Critical constraints:**
- `display: 'swap'` on EVERY font entry (FND-07, Pitfall #10, D-15)
- `cssVariable: '--font-<role>-loaded'` (suffix `-loaded` is intentional — `colors_and_type.css` already defines `--font-wordmark`/`--font-display`/`--font-body` as cascade strings; the Fonts API would overwrite if we reused the bare name. Keep the cascade in CSS so fallbacks survive font-load failure.)
- Top-level `fonts: [...]` field, NOT inside `experimental: {}` (stable in Astro 6.0)
- SWAP PATH comment for Caveat Brush is mandatory (D-17)

### `wrangler.jsonc` (config, runtime — Cloudflare edge)

**Analog:** RESEARCH.md lines 852–870 (verified against Cloudflare Workers Static Assets docs).

**Full file:**
```jsonc
{
  "$schema": "node_modules/wrangler/config-schema.json",
  "name": "studio-bluemli",
  "main": "@astrojs/cloudflare/entrypoints/server",
  "compatibility_date": "2026-05-12",
  "compatibility_flags": ["nodejs_compat"],

  "assets": {
    "directory": "./dist",
    "binding": "ASSETS",
    "run_worker_first": ["/api/*"]
  },

  "observability": {
    "enabled": true
  }
}
```

**Critical constraints:**
- `.jsonc` (not `.toml`) — current Cloudflare standard, supports comments
- `name: "studio-bluemli"` is locked by D-12 (becomes `studio-bluemli.<account>.workers.dev`)
- `main: "@astrojs/cloudflare/entrypoints/server"` — unified entrypoint; do NOT point at `./dist/_worker.js/index.js` (old pattern)
- `compatibility_date: "2026-05-12"` (today or later; do not back-date)
- `run_worker_first: ["/api/*"]` — array form with glob, NOT boolean. Reserves the `/api/*` namespace for Phase 4. Phase 1 ships NO Worker handler (any `/api/*` request 404s — acceptable; researcher noted optional 501 stub but no decision required.)
- DO NOT include `pages_build_output_dir` (Pitfall #7 — that's the legacy Pages key and means the file was copied from a 2024 tutorial)

- `<main id="main-content">` is the skip-link target.
- The container `max-width: 1200px; margin: 0 auto; padding: 0 var(--space-6)` (UI-SPEC line 234) belongs INSIDE `<main>`, either applied to `<main>` directly or via a wrapper. Mobile uses `var(--space-6)` (32px), desktop `var(--space-9)` (96px) via media query.

```tsx
import Mark from './Mark';

interface HeaderProps {

    ['/gallery',  'gallery'],
    ['/popups',   'pop-ups'],

  return (
    <header role="banner" style={{

      </a>
      <nav aria-label="Site navigation" style={{ display: 'flex', gap: 22 }}>
        {links.map(([href, label]) => (
          <a key={href} href={href}
             style={{
               fontFamily: 'var(--font-body)',
               fontWeight: active === href ? 700 : 400,
               fontSize: 16,                             // UI-SPEC Typography: nav labels are 16px Nunito 400/700
               color: active === href ? 'var(--coral-500)' : 'var(--indigo-500)',
               textDecoration: 'none',
               position: 'relative',
               paddingBottom: 4,
             }}>
            {label}
            {active === href && <Mark.Underline />}
          </a>
        ))}
      </nav>
    </header>
  );
}

export default Header;
```

**Critical deltas vs original (the 4 transforms — all are FAIL-LOUD if missed):**
1. **Transform 1:** `window.Header = Header;` → `export default Header;`
2. **Transform 2:** Add `import React from 'react';` after the eslint-disable line (defensive for SSR Fragments)
3. **Transform 3:** `src="../../assets/logo/mark.svg"` → `src="/mark.svg"` (copy mark.svg to `public/` at scaffold)
4. **Transform 4:** DELETE `backdropFilter: 'blur(4px)'` and `WebkitBackdropFilter: 'blur(4px)'` lines (CI Rule 4 will fail the build otherwise)

**Additional Phase 1 changes beyond the 4 transforms** (per UI-SPEC §"Header chrome"):
- Hash anchors (`#home`, `#gallery`) → real routes (`/`, `/gallery`, `/popups`, `/say-hi`)
- Add `role="banner"` to `<header>` and `aria-label="Site navigation"` to `<nav>`
- Active-state font weights: 400 (inactive) / 700 (active) at 16px — per UI-SPEC Typography In Use, NOT 600/800 at 14px (the original's values are pre-design-system)
- Mobile hamburger: UI-SPEC §"Header chrome" requires a hamburger button below `tablet` (640px); the design skill's Header.jsx does NOT include this. Planner must add a CSS-only responsive collapse OR (preferred) inline a `<button>` with `aria-label="Open navigation menu"` and `aria-expanded` — since there's no JS, the simplest static approach is a `<details><summary>`-based drawer or a `:target` CSS trick. **Planner decision required.**

**Active-state logic (RESEARCH.md note line 681):**
> "the active-state logic depends on the current URL, which is per-page in Astro — the component now omits the JSX-side `active` highlighting and the planner can re-introduce it as an `active` prop passed from each `.astro` page. Planner decision: pass `active` from the page."

## Shared Patterns

### Brand non-negotiables (applies to: every CSS, JSX, TSX, Astro file)

**Source:** `.claude/skills/studio-bluemli-design/colors_and_type.css` (LOCKED tokens) + UI-SPEC §"Color → Hard prohibitions" + RESEARCH.md §"CI Grep Rules — Pinned".

**Apply uniformly:**
- Background: ONLY `var(--color-bg)` (= `--cream-100`, `#F5DCC7`). Never `white`, `#fff`, `#ffffff`, `bg-white`. Only `#fff8` is whitelisted (alpha cream tint).
- Vocabulary: never `flower`, `petal`, `floral`, `bloom`, `blossom` — even in comments, alt text, file names. Earrings are "beaded clusters".
- No CSS gradients (`gradient`).
- No `backdrop-filter` / `backdropFilter` / `WebkitBackdropFilter` (the original `Header.jsx` violates this — sync Transform 4 removes it).
- No hard 1px borders (`border: 1px ...`). Use `--color-border-soft` (rgba), `--shadow-xs`, or nothing.
- Filenames under `src/pages/` are ALL LOWERCASE (Linux case-sensitive, macOS isn't).

**CI enforcement:** `scripts/check-brand-rules.sh` + `scripts/check-lowercase-filenames.sh` (both run via `.github/workflows/ci.yml`). Required status check on `main` blocks PR merge.

---

### Server-rendered React (applies to: every `src/components/design-skill/*.jsx`)

**Source:** RESEARCH.md §"Pattern 2: Server-rendered React via `@astrojs/react` (zero `client:`)".

**Apply uniformly:**
- Components are imported into `.astro` pages and rendered WITHOUT any `client:` directive. They produce HTML at build time and ship zero JS.
- Consequences: `useState`, `useEffect`, `onClick`, `onChange`, `onMouseEnter`/`onMouseLeave` are all DEAD CODE in the browser. Either strip them or accept they only run once during SSR (the initial render).
- Original design-skill components use `React.useState` for filter/hover/form state. Two specific cases need attention:

### Accessibility floor (applies to: BaseLayout + every interactive element)

**Source:** UI-SPEC §"Accessibility Floor" + REQUIREMENTS.md FND-13.

**Apply uniformly:**
- `<meta name="color-scheme" content="light">` in every page `<head>` (lives in BaseLayout)
- Skip-to-content link as the first focusable element in `<body>`; target `#main-content` on `<main>`
- `:focus-visible { outline: 2px solid var(--color-focus-ring); outline-offset: 2px; }` on every interactive element. NEVER `outline: none` without replacement.
- Landmark roles: `<header role="banner">`, `<main id="main-content">`, `<footer role="contentinfo">`, `<nav aria-label="Site navigation">`
- Touch targets ≥ 44×44px via padding (never by enlarging the visual element)
- Coral text reserved for: wordmark, h1/h2 (≥48px display font), CTA fill (cream-50 ON coral), link text, focus ring. **Never coral body prose** (contrast ~3.2:1 fails AA for normal-size text).
- Alt-text rules: mark in header `alt=""` (decorative, wordmark carries the label); mark in footer `alt="Studio Bluemli"`; never use any of the banned flower words in alt text.
