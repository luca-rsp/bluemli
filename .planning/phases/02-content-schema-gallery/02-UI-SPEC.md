## Color

Bluemli uses a **palette-driven** model: every color must be one of the named tokens (no hex hardcoding). The 60/30/10 split below maps the brand palette to the gallery surfaces specifically; the brand non-negotiable "pick three accent colors per composition, never all five" (skill rule #8) constrains the accent palette here to **coral, indigo, and lavender** βÄî olive appears only in the existing section-heading eyebrow.

### 60 / 30 / 10 split for the gallery surfaces

| Role | Token | Hex | Usage on `/gallery` + `/gallery/<slug>` |
|------|-------|-----|-----------------------------------------|
| Dominant (60%) βÄî page surface | `--cream-100` via `--color-bg` | #F5DCC7 | Page background. Cream, never white (brand non-negotiable #1). |
| Secondary (30%) βÄî text + photo | `--indigo-500` via `--color-fg` (body), `--indigo-700` via `--color-fg-strong` (price + headings), and the product photography itself | #3E4E8B / #2A3669 | Body text, price, card name. Product photos visually carry the page (brand non-negotiable: "the photo is the brand"). |
| Accent (10%) βÄî coral CTA + headline | `--coral-500` (rest), `--coral-700` (hover) | #D6553B / #A93C26 | Reserved-for list below. |

### Accent reserved for (explicit, exhaustive list βÄî no other elements may use coral on Phase 2 surfaces)

1. **Piece name headline** on the detail page (`<h1 class="name">`) βÄî `--coral-500`.
2. **Section title** above the grid (e.g., "this week's pairs") βÄî `--coral-500`, carried from existing `GalleryGrid.jsx`.
3. **CTA button background** on the detail page βÄî `--coral-500` rest, `--coral-700` hover.
4. **Back-link hover state** βÄî text transitions from `--color-fg-muted` to `--coral-500` on hover.
5. **Focus rings** (every interactive element on these surfaces) βÄî `--color-focus-ring` (= `--coral-500`), per FND-13.

Coral is **not** used for body text (brand pitfall #16 βÄî coral-on-cream contrast), for the price, for the card name, for status enum colors, or for grid-card backgrounds.

## Copywriting Contract

Voice rules (locked by design skill README): warm, casual, founder-first, sentence-case, friendly parentheticals, no emoji. The only decorative glyph permitted is `βôΓ`/`βôΞ` in coral. Never write "under $100" or any price-ceiling claim.

### Primary CTAs (detail page βÄî copy varies by status)

| State | CTA button copy | Target |
|-------|-----------------|--------|
| `available` / `one-of-one` / `reserved` | **Ask about this pair on Instagram** | `https://ig.me/m/studiobluemli` |
| `sold` | **This pair sold βÄî DM me about something similar** | `https://ig.me/m/studiobluemli` |

The CTA is always a real `<a href>` link (works with JS disabled βÄî progressive enhancement, CON-10 pattern, applied early here for consistency). Mailto fallback below the button, every state:

> or email [hi@studiobluemli.com](mailto:hi@studiobluemli.com)

### Back navigation (detail page βÄî appears twice, top and bottom)

> βÜê Back to the gallery

ALL CAPS via `text-transform`, with the arrow as a literal `βÜê` character (U+2190, not an emoji).

### Status labels (rendered identically on grid card meta and detail-page status badge)

| Enum | Label |
|------|-------|
| `available` | Available |
| `sold` | Sold |
| `one-of-one` | One of one |
| `reserved` | Reserved |

Sentence case. No "SOLD OUT!" caps-yelling. No exclamation marks. Eyebrow letter-spacing + ALL-CAPS handled at the CSS layer (`text-transform: uppercase` + `--ls-caps`), not by writing the label in caps in the markup.

### Empty state (`/gallery`)

**When zero pieces exist in the `gallery` collection** (would only occur if all 6 seed pieces are removed βÄî defensively included for content-collection safety):

| Element | Copy |
|---------|------|
| Heading | the studio is between pairs right now |
| Body | new pieces drop after each pop-up βÄî follow on [Instagram](https://instagram.com/studiobluemli) (`@studiobluemli`) to catch the next one |

Sentence case throughout, friendly parenthetical, lowercase Instagram handle styled consistently with the brand voice. **No founder face**, no founder name on this surface. (D-07's brand-voiced placeholders ship 6 real pieces so this state is not expected to render in Phase 2 preview; the copy exists as a defensive contract.)

### Error state (build-time only βÄî Phase 2 has no runtime errors on these surfaces)

Phase 2's gallery surfaces are **prerendered**. There is no runtime fetch, no client-side state, no form. The only "error" path is **a Zod schema violation at build time** (SC2 βÄî typo'd frontmatter fails the build), which surfaces in the CI build log to the engineer, not to a visitor. No user-facing runtime error UI is in scope for this phase.

If an `<img>` 404s in the browser (variant missing from `public/gallery/<slug>/`), the photo falls back to the `--cream-200` solid background declared on the `<img>` rule (per the sketch-findings reference). No alt-text-only fallback message is shown βÄî the prebuild pipeline guarantees variants exist; a missing variant is a build bug, not a content bug.

### Destructive confirmation

| Action | Confirmation copy |
|--------|-------------------|
| not applicable | **No destructive actions in Phase 2.** Founder workflow: flip `status: available` βÜ£ `status: sold` in `index.md`, never delete the folder (CNT-10, D-11). CONTENT_EDITING.md (CNT-12) ships a clearly labeled "never delete, flip availability" section. |

### Alt text contract for product photography

- Default alt text for `<img class="product-photo">` (detail page) and `<img class="card-photo">` (grid): `alt={piece.name}` βÄî e.g., `alt="Coral cluster"`.
- **Forbidden** in any alt text: `flower`, `petal`, `floral`, `bloom`, `blossom` (FND-10 + PAG-09 βÄî enforced by CI grep).
- Per-piece custom alt is a v1.x schema field (deferred per CONTEXT.md `<deferred>`).

### Placeholder seed descriptions (6 pieces, written by Phase 2 executor)

Each `src/content/gallery/<slug>/index.md` ships with a 1-sentence brand-voice description per D-07. Voice constraints (must pass brand-check Rule 2 + Rule 7):

- Sentence case. No "SOLD OUT" caps. No exclamation marks.
- Allowed vocabulary for color/material: bead, cluster, glass, shimmer, sparkle, pair, cluster-drop, sway, palette names from the brand (cream, coral, indigo, lavender, sage, olive, saffron, cobalt, mustard).
- **Forbidden vocabulary:** flower, petal, floral, bloom, blossom (CI-enforced); "under $100" / any price-ceiling claim (skill rule #6); "SOLD OUT", "Buy Now", "Featured in" (brand anti-features).
- One sentence per piece. Friendly parenthetical optional. No emoji. No `βôΓ`/`βôΞ` in seed descriptions (decorative glyph is brand-mark-only; descriptions stay quiet).

The exact 6 placeholder strings are the executor's discretion (locked-in voice rules above) βÄî sample for "Coral cluster": `bright coral and cream glass beads, hand-clustered into a pair that wears every day`.

## Performance / image contract

- **No client JS.** Both gallery surfaces are server-rendered Astro pages with `prerender = true`. The grid uses React JSX (`GalleryGrid.jsx`) rendered as static HTML βÄî no `client:` directive (FND-01).
- **Image strategy** (FND-12 + LCH-05 Lighthouse mobile βâΞ 90):
  - `<img>` tags, not Astro `<Image>` component (because `passthroughImageService()` doesn't transform; pre-optimized WebPs are emitted by the prebuild pipeline D-01).
  - All product images include `width` + `height` attributes matching the variant's native dimensions (preventing CLS).
  - `loading="lazy"` on grid cards below the fold; **NOT** `loading="lazy"` on the detail-page hero photo (it's the LCP element).
  - `decoding="async"` on all product images.
- **Per-piece `og:image`** (D-12, CNT-09): absolute URL pointing at `https://studiobluemli.com/gallery/<slug>/hero-800.webp` (or the preview hostname during PR deploys βÄî planner picks how to resolve the base URL at build time). Emitted as `<meta property="og:image">` in the detail page's `<head>` (either inline or via a minimal `<SEO />` component βÄî planner's discretion per CONTEXT.md `<decisions>`).
