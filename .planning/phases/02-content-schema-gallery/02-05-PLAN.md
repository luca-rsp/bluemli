3. **Adding a new gallery piece (step-by-step)**:
   1. Navigate to the `src/content/gallery/` folder on GitHub.com.
   2. Click "Add file → Create new file".
   3. In the filename box, type your slug followed by `/index.md` — for example **`cluster-cobalt/index.md`** (LOW-2 replaces previous `cluster-rose` example). The slash automatically creates the folder.
   4. Paste the frontmatter template from the reference card.
   5. Fill in values: name, price, status, description, featured, published_at (today's date YYYY-MM-DD).
   6. Choose "Create a new branch", name it like `add-cluster-cobalt`.
   7. "Propose new file" → "Create pull request".
   8. Wait ~3 minutes for the preview URL in the PR.
   9. Merge.

   Note: screenshots will be added once the founder confirms preferred GitHub-website theme.

4. **Uploading the photo**:
   1. Navigate to your new folder (e.g., `src/content/gallery/cluster-cobalt/`).

Capital letters work too — `Hero.HEIC` is fine; the build is case-insensitive.
   5. Choose the same branch as your `index.md`.

5. **Editing an existing piece** — Click the pencil icon on the piece's `index.md`, edit, branch, PR, preview, merge.
