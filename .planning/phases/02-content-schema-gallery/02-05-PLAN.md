1. **Overview** — One paragraph: "How to add or update gallery pieces and pop-ups using GitHub's website. No code knowledge needed. Every change opens a pull request, which gives you a preview URL to check before merging."

2. **What goes where (file map)** — Brief table:
   - Gallery pieces: `src/content/gallery/<piece-slug>/` (one folder per piece, contains `index.md` and `hero.heic` or similar)
   - Pop-ups: `src/content/popups/YYYY-MM-DD-<event-slug>.md` (one file per event, date-prefixed)
   - Site-wide settings (tagline, IG handle): `src/content/site/config.md`

3. **Adding a new gallery piece (step-by-step)** — Walkthrough using the GitHub website:
   1. Navigate to the `src/content/gallery/` folder on GitHub.com.
   2. Click "Add file → Create new file".
   3. In the filename box, type your slug followed by `/index.md` — for example `cluster-rose/index.md`. The slash automatically creates the folder.
   4. Paste the frontmatter template (provided below in §Frontmatter reference card).
   5. Fill in the values: name, price, status (`available`, `sold`, `one-of-one`, or `reserved`), description, featured, published_at (set this to today's date — YYYY-MM-DD).
   6. Scroll down and choose "Create a new branch for this commit" — give the branch a short name like `add-cluster-rose`.
   7. Click "Propose new file" → "Create pull request".
   8. Wait ~3 minutes. A preview URL will appear in the pull request — click it to see your new piece.
   9. If everything looks right, click "Merge pull request". The piece is now live on the main site.

   Note for screenshots: this section will include screenshots once the founder confirms her preferred GitHub-website appearance (light/dark theme). For now, the prose alone is the source of truth.

4. **Uploading the photo** — Same GitHub-website flow:
   1. Navigate to your new folder (e.g., `src/content/gallery/cluster-rose/`).
   2. Click "Add file → Upload files".
   3. Drag the photo from your computer (or your phone, if you're on the GitHub mobile site). HEIC works — no need to convert.
   4. Rename it to `hero.heic` (or `hero.jpg` if it's a JPG) before clicking "Commit changes".
   5. Choose the same branch as your `index.md` so they land in the same pull request.

5. **Editing an existing piece** — Click the pencil icon on the piece's `index.md` on GitHub, edit, "Create a new branch", "Propose changes", click the preview URL in the PR, merge. (Same flow as adding, fewer steps.)

6. **Marking a piece as sold** — Open the piece's `index.md`. Change `status: available` to `status: sold`. Save → branch → PR → preview → merge. The piece stays in the gallery with a quiet "Sold" label.

7. **## Never delete a piece (flip availability instead)** — One-of-one pieces sometimes sell out, but they're still part of the studio's portfolio archive. Deleting a folder removes the piece from the site permanently. **Always flip the status to `sold` instead of deleting.** If a piece was always one-of-a-kind, use `status: one-of-one` so the label says "One of one" instead of "Available". Same for `reserved` — that's a soft hold, never a deletion.

8. **Adding a pop-up event** — In `src/content/popups/`, "Add file → Create new file". Filename: `YYYY-MM-DD-event-name.md` (e.g., `2026-06-15-nopa-block-party.md`). Paste the pop-up frontmatter template (below). Fill in date, start_time, end_time, location, etc. Branch → PR → preview → merge.
